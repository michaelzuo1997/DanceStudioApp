# DB Fix Session - 2026-02-19

## Issues Fixed

### 1. No Dance Classes Showing
**Root Cause:** RLS policy only allowed `authenticated` users to read `CLASSES` table.

**Fix:** Added `anon` read policies to:
- `CLASSES`
- `class_timetable`
- `class_bundles`

### 2. Balance Top-up Not Working
**Root Cause:** `topup_balance` RPC had poor error handling and wrong return type.

**Fix:** 
- Changed return type to JSON (`{ ok, new_balance?, error? }`)
- Added user info auto-creation
- Added proper validation

### 3. Migration Errors (column does not exist)
**Root Cause:** Deployed `CLASSES` table was missing multiple columns (`name`, `class_type`, `start_time`, `end_time`, etc.)

**Fix:** Added comprehensive column existence checks before any operations.

## DB Schema Reference

See `DB_SCHEMA.md` for complete schema documentation.

### Key Tables
- `CLASSES` - Individual class sessions
- `Users Info` - User profiles & balance
- `enrollments` - Class bookings
- `transactions` - Payment history
- `class_categories` - Dance types
- `class_timetable` - Weekly schedule
- `class_bundles` - Class pass packages

### RPC Functions
- `topup_balance(p_amount)` → JSON
- `purchase_bundle(p_bundle_id)` → TABLE
- `cancel_enrollment(p_enrollment_id)` → TABLE
